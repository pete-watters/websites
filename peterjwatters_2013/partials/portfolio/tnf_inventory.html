3.1 What is TNFMON DRE?

TNFMON DRE or TNFMON DCV replacement Engine is an internal tool for use by TNF employees to analyse information gathered from customers. There are two main types of information gathered from customer sites:

-          Statistical performance information

-          Inventory information

This information is taken from SourceWorks probes and imported to a database at TNF HQ in Sandyford allowing it to be manipulated in several ways.

  

3.2        High Level Functionality

TNFMON DRE can be split into three main areas:

1.       Statistical Analysis tool

·     Analysis of Sourceworks server performance

·     Generation of performance reports on a per server basis

·     Proactive monitoring of performance and issue troubleshooting

 

2.        Customer Inventory

·     Information about software and hardware configuration

·     Back up of configuration files and scripts

 

3.       Customer Matrix

·     Trending and mining of customer wide data

·     Prediction of future growth based on previous trends

 

Using TNFMON DRE SourceWorks server performance can be monitored in a fast and efficient manner. Hardware setup and configuration settings are backed up and centralised at TNF head offices and customer information can be mined and trended to help make key decisions.

Currently the main focus of this project has been to provide an efficient way of viewing performance statistics that can replace the historically used method of running a DCV on-site. Running a DCV on-site can be quite cumbersome and time consuming and when investigating critical issues sufficient time to generate and analyse these reports is simply not available.

To gather these statistics and analyse them in an efficient manner a well-designed backend database was needed and once this had been created for the statistical analysis tool it created a great base to extend this project further, elaborating on existing features and functions and adding new ones. Essentially by collecting the inventory and other customer information the data stored on the DB is building up and new features can be added for different analysis metrics that are of use to different departments.

 

 

3.3        TNFMON DRE Technologies Used

TNFMON is a Java Enterprise Edition (J2EE) web application which relies on a variety of technologies to provide the look and feel of a rich internet application (RIA) without any of the usual associated dependencies on flash or other technologies. This means that it is portable and independent and can run on any machine or device and in any browser.  The main technologies used were:

1.       Java: Java Servlets were used for the backend logic and Java Server Pages were used to build the front end in a  dynamic way

2.       JavaScript: Javascript was used in many forms and flavours in this application. The main use cases were the use of the jQuery UI library, jQuery itself and HighCharts charting API.

3.       HTML5/CSS3: HTML5 and CSS3 were used to piece together the front end

3.4         TNFMON DRE Technologies Overview

Some of the aforementioned technologies may be new to a lot of people so this section gives a quick overview of what each does and where to find more information about them.

3.4.1        jQuery

jQuery is an open source JavaScript library that enables developers to make use of hundreds of functions and plug-ins and was designed to change the way developers write JavaScript. The project began in 2005 and has been a huge success since then with many developers adding to the ever increasing plug-in library. It simplifies the core features JavaScript is commonly used for such as event handling, AJAX interactions and animations. Many developers have struggled with the same problems in their application and jQuery provides an easy way to reference a set of predefined functions and UI tools.

There are thousands of jQuery plug-ins available for download to serve any purpose needed with new ones being added daily. To use jQuery all that is required is the download of the source JavaScript file and inclusion in your HTML page. This then enables the use of the core functionality and additional files can be included for different plug-ins as needed. All that is needed to use jQuery is this single JavaScript file and once referenced in your web page you can begin to utilize it as an engine and write your application rapidly.

jQuery makes developing AJAX applications far simpler and their tag line is even “jQuery, the write less, do  more Java Script Library” which sums up what the project is all about. You can find out more about jQuery on their homepage at www.jquery.org .

 

3.4.2        jQuery UI

The jQuery user-interface (UI) library is used extensively in the front end design of this project and provides developers with a set of excellent tools to aid them in user interface design. A custom built package can be downloaded to suit your exact needs and can be themed using one of the pre-defined themes or a bespoke theme can be developed using the ThemeRoller which allows users to develop a custom theme and preview before downloading.

Some of the best built in features are the accordion, tabs and dialog all of which are used extensively in this project. These features are excellent for helping build a dynamic web page using the DOM and div tags effectively to minimise the amount of times a user needs to change page. In historical web applications links would more often than not bring you to a new page which would require loading. Using the jQuery UI developers can easily organise their page into clear sections using <div> tag’s and can offer a lot of functionality without requiring the user to re-load the page.

Along with these layout features there is a vast amount of other AJAX animation features available to really help bring applications to life. Complex interactions such as dragging, dropping and resizing elements can be performed along with various animations such as colour animation, fading in and out of div elements, hiding elements and many more. With a bit of work initially a highly interactive and cross browser compatible AJAX web application can be developed and this is why it was chosen for this project.

An excellent resource for getting started with jQuery is to try out the demo pages at www.jquery.org or to use the excellent guide “jQuery Novice to Ninja” which provides an abundance of useful tips when getting started with jQuery. Time spent getting to understand how it works will reap great benefits in future and development time of other projects will be significantly reduiced.

 

3.4.3        HighCharts

One of the best freely available chart generation and plotting solutions is the HighCharts library (www.highcharts.com). This is a pure JavaScript library, freely available for plotting charts without the need for any internet access or browser plug-ins. All that’s needed is the inclusion of a JavaScript file anywhere on the web server and this can be referenced to plot graphs anywhere in the application. Another key advantage is that it’s cross browser and device compatible. The charts are rendered using the HTML5 canvas tag in most browsers and using VML in Internet Explorer as this does not as yet fully support HTML5 and the canvas tag.

No client side plug-ins are needed which breaks from dependence on Flash or Silverlight. All that’s needed to render HighCharts is the HighCharts core java script file and either the jQuery or MooTools framework (both are common web development java script libraries).

There is a huge library of charts including some very interesting combination charts like the chart below which comprises of a bar chart with a trend line and an embedded pie chart. HighCharts allow the developer to draw charts with numerous axes and plot several different chart types over one another which can really help convey as many trends as possible on one chart. A bespoke and customised chart was written specifically for TNFMON DRE to provide the exact functionality needed.

 

3.5        Client Requirements

The client computer should have any browser supporting JavaScript. For a better user experience it is advised to use a W3C Standard Compliant browser (e.g. Firefox, Safari, Chrome, Opera) as this will ensure HTML5 and CSS3 compatibility.

4.2        Statistical Analysis Tool:

4.2.1        Plotting Statistics:

To plot statistical information you need to provide input parameters to the form. The statistics drop down is generated by the server so only sites on the system will appear here.  The first step is to choose an OPCO site using the drop down. This drop down utilises a jQuery plugin to allow it to filter based on user input and provide a search function.

Figure 4‑2 Statistics Search Drop down

Once you have chosen a host the next step is choosing a date or date range. If you wish to plot for a single day simply choose the date that you wish to search for.  Currently the time range used is 00:00 on the start date to 23:59 on the end date however this can be changed in future to have this as a search parameter.

You can type in the required date however using the selector is recommended to ensure the format is correct:

Figure 4‑3 Date range selector

There are other options to choose from including options to plot the month to date or the previous month however these may take longer to load due to the higher volume of data being gathered. This date input selector was created using a jQuery plugin that was customized and extended to provide the best options for this project.

Once the hosts and dates have been chosen the search button should be clicked to initialize the search. The enter key has been disabled as a submitter as this can cause issues as it can bypass javascript validation.

 

4.2.2        Statistical Search: Data Found:

If data matching the input parameters has been found the page will generate the graphs and inventory for that server for the date range provided the page will refresh with 2 new tabs visible for Statistics and Inventory. The time taken to generate these tabs depends on the range chosen however a ‘Loading’ message will show up in the meantime to inform the user it’s gathering the information. Originally the system was designed and tested to provide a report of performance for a single day similar to the old DCV so multiple day generation is possible but not yet optimal.

4.2.4        Using the graphs:

Several SourceWorks performance metrics are plotted and the graphs available are based on the most used graphs from the original DCV. The way the system is setup allows this to be extended in future as all statistical data is available in the database and simply needs to be graphed.

The flow of the application is to generate a JSON string of the graph values in JavaScript. Once this is generated for a metric, for example total NIC speed, this data series is available to plot on any graph against any other metric. This makes the system very extensible and future proof for any additional statistics needed.  An additional feature available in TNFMON DRE is that series are plotted on multiple axes, in fact up to 5 are available.

This means that difference series types can be compared more easily and effectively. For example the user can compare percentage packet loss with NIC speed in MB/s and the count of TDRs logged. In excel it is only possible to plot on 2 different axes so this is a huge step forward. An overview of the graphs themselves and how to best utilize the features is given below to help users understand the full power available to them.

To plot the inventory per OPCO simply browse to the inventory search section and choose an OPCO from the drop down menu and press search.

The customer matrix section allows users to view customer information on a global basis and to tend customer data. This section was added as an overview of what can be done using the underlying database and it was intended to elaborate and extend on this. For now it’s quite simple and basic with just a few sections.

4.3.4.1         Customer Information Table

The customer information table is a dynamically updating table containing details about all customers with information in the database. It is built with a jQuery data table plugin which allows the user to filter the table based on keywords. The search parses any field in the table for instances of this string and provides the user with a fast and efficient way of analyzing customer data. Using a key-word search rather than a filter leaves it open to more options. The user could for example type in H2 to find a list of all sites with a H2 probe installed. Multiple keywords are also accepted separated by a whitespace.

Figure 4‑28 Customer Information Table

The row grouping is as follows:

-           Customer: The OPCO long name

-           Site: The site in question

-           Probe version: The Kernel version with build number

-           Card Type: The NIC card type short and long name

-           Card ports: The count of input ports on the installed card

-           Peak data rate: The peak data rate for that site (Max NIC Speed)

-           Sourceworks Version

-           Feeds: the record output types e.g. CEM, SUMO, XMINE etc.
